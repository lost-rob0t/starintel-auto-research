- [[file:../../design/star-server/QUASAR-002-clog-hybrid-prototype.org][QUASAR-002 CLOG Hybrid Prototype Architecture]]
- [[file:../../design/star-server/QUASAR-003-graph-document-command-protocol.org][QUASAR-003 Graph Document and Command Protocol]] — blocked on prototype evidence

Quasar is a basic typed graph editor first. Its core is graph-document editing, layout, persistence, import, export, undo, and typed node and edge operations. Quasar extensions and external integrations own optional actions and transforms. Star-Lang may contribute compatible type libraries, document specifications, predicates, relations, identifiers, and import or export contracts, but it does not provide Quasar transforms. Domain servers, distributed actors, evidence workflows, and large-scale projections are optional integrations rather than prerequisites for the editor.

The selected prototype baseline is now SBCL, Common Lisp, CLOG, and a hybrid Cytoscape.js composite component. CLOG owns browser bootstrap and ordinary UI controls while Cytoscape retains high-frequency graph interaction in the browser. The canonical graph, graph operations, undo, action discovery, action execution, and persistence remain in a CLOG-independent Common Lisp core. React, TypeScript, Tauri, and a formal public graph protocol are deferred until prototype evidence shows they are required. The earlier TypeScript client research remains an extraction option rather than the immediate implementation baseline. The reserved protocol design is =QUASAR-003= and remains blocked until manual edits, batches, undo, persistence, session reconstruction, and one action have been exercised.

* Language and Execution Research

- [[file:../star-lang/STAR-LANG-INDEX-000-language-research.org][STAR-LANG-INDEX-000 Star-Lang Language Research]]
- [[file:../../research/star-lang/STAR-LANG-RESEARCH-000-language-design-foundations.org][STAR-LANG-RESEARCH-000 Star-Lang Language Design Foundations]]
- [[file:../../research/star-lang/STAR-LANG-RESEARCH-001-common-lisp-comparative-baseline.org][STAR-LANG-RESEARCH-001 Common Lisp Comparative Baseline]]
- [[file:../../research/star-lang/STAR-LANG-RESEARCH-006-specification-libraries-and-domain-servers.org][STAR-LANG-RESEARCH-006 Specification Libraries and Domain Servers]]

The completed Star-Lang foundation defines a bounded declarative language for durable analysis runs, explicit actor effects, Prolog capability routing, checkpoint replay, and declared condition and restart handling. The active comparison loop begins with a Common Lisp baseline but must complete equivalent Racket and Scheme prototypes before recommending a language.

The specification/domain loop defines digest-locked data-only libraries, additive typed-document extension, explicit source/predicate/destination relations, immutable attached datasets, actor contracts, and keyed domain-server manifests. Domain servers are the Star-Lang declaration layer over cl-gserver/Sento managed components; they own bounded aggregate authority and indexes rather than creating one actor per document. A separate numbered design must still freeze interfaces before implementation begins.

* Footnotes and Glossary

[fn:project-index] Project index: A maintained navigation file that lists the authoritative notes, active work, dependencies, and research order for one project area.

[fn:starintel] StarIntel: The local-first intelligence platform being designed in this repository to collect, normalize, connect, search, analyze, and export structured documents through actor-based components.

[fn:canonical-inventory] Canonical inventory: The primary list that remains authoritative for what ideas exist, even when individual ideas are copied into more focused research or design files.

[fn:research-loop] Research loop: One bounded cycle that selects a single topic, performs deep research, records findings and sources, provides an implementation-oriented example, and identifies the next design decision.
