* TODO Objective

Use Emacs Lisp as the reference implementation for research capture, design-file creation, SRFC management, Org-roam indexing, Org-ql views, context selection, validation, implementation promotion, graph export, and local Git workflow.

* TODO Workflow

** TODO Setup

Run =star/research-setup= to create the repository directories and configure Org-roam beneath =roam/=.

** TODO Capture

Create numbered research, design, and SRFC documents with the =star/research-new-*= commands.

** TODO Context Selection

Agents receive a generated context bundle containing only the active file, its direct local file links, and its nearest project index. The bundle must remain below =star/research-context-max-bytes=.

** TODO Review

Use Org-ql views to find active, blocked, project-specific, stale, and unreviewed work.

** TODO Promotion

Copy exactly one approved design into =roam/implement/= with =star/research-promote-design=.

** TODO Validation

Validate headers, numbered filenames, required headings, and version tables before staging or committing.

** TODO Publication

Export Org-roam nodes and edges into a stable graph JSON document consumed by Emacs, the web UI, and the desktop graph application.

** TODO Agent Access Rule

Do not recursively concatenate the tree. Direct links and indexes define the permitted context frontier.

** TODO Data Ownership

Org files are authoritative. The Org-roam database and exported graph JSON are derived state.

** TODO IDE Contract

All IDE surfaces consume the same node, link, project, status, citation, actor, dataset, and implementation relationships. Writes must route through validated workflow operations.
