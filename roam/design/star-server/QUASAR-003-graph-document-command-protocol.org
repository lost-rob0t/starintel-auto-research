* Status

This design is intentionally blocked until the CLOG hybrid prototype has exercised:

- manual node and edge mutations;
- committed node movement;
- inspector edits;
- graph batches;
- transaction-level undo and redo;
- save and reopen;
- browser refresh and UI-session reconstruction;
- one in-process Common Lisp action.

The earlier plan assigned protocol work to =QUASAR-002=. The CLOG prototype decision moved that work to =QUASAR-003= so the protocol is derived from observed editor operations rather than guessed before implementation.

* Required Evidence

Before this design becomes active, the prototype should record:

- the actual graph operation classes used by the core;
- which renderer events become durable operations;
- the minimum data required to rebuild a UI session;
- the graph snapshot fields required for lossless save and reopen;
- the batch structure required by actions;
- the inverse data required by undo;
- whether local command identifiers and revisions provided practical value;
- reconnect behavior observed through CLOG;
- whether a separate client remains desirable.

* Intended Scope

When unblocked, this design should freeze:

- canonical graph-document JSON;
- canonical graph-batch JSON;
- node, edge, metadata, position, viewport, and type-reference schemas;
- graph operation and command envelopes;
- command identifiers where justified;
- revision rules;
- batch atomicity;
- error and rejection representations;
- inverse-command requirements;
- snapshot and command-log boundaries;
- reconnect and resynchronization behavior;
- Common Lisp bindings;
- TypeScript bindings only if an independent client is selected;
- conformance fixtures.

* Non-Goal While Blocked

No implementation should create a public remote protocol merely to satisfy this reserved design number.
