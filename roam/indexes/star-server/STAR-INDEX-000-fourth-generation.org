* Research Loop

The research loop[fn:research-loop] follows the order recorded in the canonical idea backlog unless a dependency requires a different order.

** DONE Loop 001: Other OSINT Project Features

- [[file:../../research/star-server/STAR-RESEARCH-001-osint-feature-landscape.org][STAR-RESEARCH-001 OSINT Feature Landscape]]

This loop compared mature intelligence and open-source intelligence platforms, extracted reusable product patterns, mapped those patterns onto the current StarIntel architecture, and proposed =STAR-SERVER-001 Capability Catalog and Playbook Runtime=.

Loop 001 closed when nsaspy explicitly instructed the workflow to continue to the next loop on 2026-07-19. The proposed design remains unimplemented and retains its own approval requirement.

** REVIEW Loop 002: Collection Actors

- [[file:../../research/star-server/STAR-RESEARCH-002-collection-actor-architecture.org][STAR-RESEARCH-002 Collection Actor Architecture]]
- [[file:../../design/star-server/STAR-COLLECT-001-collection-actor-contract-source-runtime.org][STAR-COLLECT-001 Collection Actor Contract and Source Runtime]]

This loop classifies the large source inventory into reusable execution classes, defines source instances, policy snapshots, collection runs, source-specific checkpoints, atomic collection batches, and external-worker boundaries, and proposes a common collection runtime before platform-specific adapters.

The next queued area is section 3 of the canonical idea backlog: document metadata expansion, unstructured documents, custom view code, attachments, files, and expert-system support. It must not begin until Loop 002 is reviewed or explicitly superseded.

* Footnotes and Glossary

[fn:project-index] Project index: A maintained navigation file that lists the authoritative notes, active work, dependencies, and research order for one project area.

[fn:starintel] StarIntel: The local-first intelligence platform being designed in this repository to collect, normalize, connect, search, analyze, and export structured documents through actor-based components.

[fn:canonical-inventory] Canonical inventory: The primary list that remains authoritative for what ideas exist, even when individual ideas are copied into more focused research or design files.

[fn:research-loop] Research loop: One bounded cycle that selects a single topic, performs deep research, records findings and sources, provides an implementation-oriented example, and identifies the next design decision.
