* Purpose

This repository is the Emacs-first control plane for Starintel research, design files, SRFCs, implementation promotion, and graph-oriented IDE surfaces.

The Org tree under =roam/= is the source of truth. Elisp under =lisp/= owns capture, indexing, validation, bounded context construction, design promotion, Git workflow helpers, and graph export.

* Repository Shape

#+begin_example
lisp/                         Emacs workflow implementation
roam/design/                  Numbered design files and SRFCs
roam/research/                Research trackers, inbox captures, and findings
roam/implement/               Designs promoted for implementation
roam/indexes/                 Roadmaps and project indexes
test/                         ERT tests
ui/web/                       Web graph IDE
ui/desktop/                   Desktop graph IDE
#+end_example

* Dependencies

- Emacs 29 or newer.
- Org mode.
- Org-roam for graph indexing and capture.
- Org-ql for dashboard queries.
- =simple-httpd= only when serving the web UI directly from Emacs.

* Doom Emacs Setup

#+begin_src emacs-lisp

* Workflow

** TODO Capture

Use Org-roam inbox capture for raw material. Convert accepted design notes into numbered files with =star/research-new-design=.

** TODO Research

Track status, sources, repositories, decisions, rejected approaches, implementation issues, and next action in Org properties and TODO states.

** TODO Context

Do not load the entire tree into an agent context. =star/research-build-context-bundle= includes only:

- The active Org file.
- Its directly linked local files.
- One-level manifests for directly linked directories.
- The nearest project index, with the roadmap as fallback.

The result is truncated to =star/research-context-max-bytes= and written under =.starintel/context/=.

** TODO Design

Create design files under =roam/design/<project>/=. Numbering, headers, version table, required headings, and citations are validated before Git staging.

** TODO Implementation

Use =star/research-promote-design= to copy exactly one approved design into the matching path below =roam/implement/=.

** TODO Git

Use the =C-c s g= commands. Staging is limited to =lisp/=, =roam/=, =ui/=, =README.org=, and =.dir-locals.el=. Generated databases, context bundles, and graph exports remain untracked.

** TODO IDEs

Emacs is the reference workflow engine. The web UI and desktop graph app consume the same JSON graph export and must route writes through validated workflow operations.
