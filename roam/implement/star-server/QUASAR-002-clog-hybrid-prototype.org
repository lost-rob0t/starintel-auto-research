* Active Slice

This implementation is limited to the first prototype checkpoint:

1. loadable =quasar/core= and =quasar/ui-clog= systems;
2. a renderer-independent Common Lisp graph model;
3. a CLOG[fn:clog] workbench shell;
4. one Cytoscape.js[fn:cytoscape] composite graph component;
5. one browser-to-Lisp =rendererReady= signal;
6. projection of two nodes and one edge from the read-evaluate-print loop[fn:repl];
7. dependency-light core tests.

Editing gestures, action execution, persistence, Star-Lang integration, remote services, and the public Quasar protocol remain outside this checkpoint.

* Footnotes and Glossary

[fn:clog] CLOG: Common Lisp Omnificent GUI, a Common Lisp framework that uses a browser for rendering and WebSockets for continued communication with Lisp.

[fn:cytoscape] Cytoscape.js: A JavaScript library that renders and analyzes node-and-edge graphs in a web browser.

[fn:repl] Read-evaluate-print loop: An interactive programming prompt that reads an expression, evaluates it, prints the result, and repeats.

[fn:sbcl] SBCL: Steel Bank Common Lisp, a widely used compiler and runtime for the Common Lisp programming language.

[fn:asdf] ASDF: Another System Definition Facility, the standard Common Lisp tool used to define, load, and test software systems and their dependencies.

[fn:ci] Continuous integration: An automated service that checks a proposed software change by installing its dependencies and running its build and tests in a clean environment.
