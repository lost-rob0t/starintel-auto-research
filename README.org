#+title: Starintel Auto Research

* Purpose

This repository is the Emacs-first control plane for Starintel research, design files, SRFCs, implementation promotion, and graph-oriented IDE surfaces.

The Org tree under =roam/= is the source of truth. Elisp under =lisp/= owns capture, indexing, validation, context bundling, Git workflow helpers, Org-roam database operations, and GitHub Pages export.

Published second brain: [[https://lost-rob0t.github.io/starintel-auto-research/][lost-rob0t.github.io/starintel-auto-research]]

* Org-roam Second Brain

Load =lisp/starintel/second-brain.el= from Emacs, then use:

#+begin_example
M-x star/roam
M-x star/roam-capture
M-x star/roam-sync
M-x star/pages-build
M-x star/pages-serve
M-x star/pages-open
M-x star/pages-stop
#+end_example

The repository =.dir-locals.el= points Org-roam at =roam/= and stores its generated database under =.cache/=.

Build and serve the same site used by GitHub Pages:

#+begin_src sh
bash scripts/publish-pages
python3 scripts/check-pages-links.py _site
python3 scripts/serve-pages
#+end_src

Open [[http://127.0.0.1:8042/][127.0.0.1:8042]] after starting the server. Do not open =_site/index.html= directly through a =file://= URL: browsers block the JSON requests used by search and the graph on local files.

The export uses a temporary normalized copy of the Org tree, builds the Org-roam database, exports one HTML page per Org file, adds backlinks, creates search and graph indexes, and checks every internal page, asset, and anchor link. GitHub Actions also tests the generated site through HTTP and verifies the public Pages URL after deployment.

See [[file:roam/indexes/second-brain/SECOND-BRAIN-000-org-roam-pages.org][SECOND-BRAIN-000 Org-roam Pages]].

* Rule

Do not load the entire research tree into an agent context. Build a bounded context bundle from the active node, its direct links, and its project index.
